What It Generates & Design Decisions

ServingRuntime

Shared - one per blueprint

  • RHOAI vLLM image
  • HF token + env config
  • Dashboard annotations

InferenceService

One per model

  • --model <HuggingFace ID>
  • GPU + tensor-parallel-size
  • RawDeployment mode

Design Decisions

Additive Only

Never removes or wraps existing NIM/API paths. All generated resources are new conditional branches.

Disabled by Default

vllm.models.<model>.enabled: false in values.yaml. Users opt in explicitly per model at deploy time.

Runtime Compatibility

Fetches registry.py from the exact deployed vLLM version to verify model architecture support - not static docs.

Secret Reuse

Detects existing HF token secrets and references them. Only generates a fallback Secret when none exist.

NIM Serving Handoff

Incompatible models don't dead-end. User chooses NIM serving or keep current. If NIM: skill asks, then invokes /bp-add-nim-serving via Skill tool with a soft preference note.

Skill Chaining

Each skill owns its full flow. vLLM passes a soft handoff note via $ARGUMENTS — NIM still rediscovers and confirms, but biases toward the user's earlier choice.
